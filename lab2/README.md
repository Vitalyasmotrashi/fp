## Ключевые концепции этой лабы

### Функции высшего порядка (Higher-Order Functions)

Это главная тема лабы. Функции высшего порядка — это функции, которые принимают
другие функции как аргументы или возвращают функции как результат.

В Erlang функция — это полноценное значение, её можно:
- передать как аргумент: `takewhile(fun(X) -> X > 0 end, List)`
- сохранить в переменную: `F = fun(X) -> X * 2 end`
- вернуть из другой функции: `iterate` возвращает `fun(X) -> ... end`
- вызвать через переменную: `F(5)` → `10`

### Анонимные функции (fun-expressions)

```erlang
fun(X) -> X + 1 end          % принимает 1 аргумент
fun(X, Y) -> X + Y end       % принимает 2 аргумента
fun(0) -> zero; (N) -> nonzero end  % с pattern matching
```

Переменные из окружающего контекста «захватываются» (замыкание/closure):
```erlang
N = 10,
F = fun(X) -> X + N end,  % N захвачено из контекста
F(5).  % => 15
```

## Задание 1: `list_heads/1` — файл `lab2_task1.erl`

### Код

```erlang
list_heads([]) ->
    [];
list_heads([[H | _] | Tail]) ->
    [H | list_heads(Tail)];
list_heads([_ | Tail]) ->
    list_heads(Tail).
```

### Объяснение

**Задача:** взять список, который может содержать что угодно (числа, атомы, кортежи,
вложенные списки), и вернуть список из **первых элементов** только тех элементов,
которые являются **непустыми списками**.

Три варианта функции, разобранных по образцам:

1. `list_heads([])` — внешний список пуст → `[]`

2. `list_heads([[H | _] | Tail])` — самый важный случай:
   - Текущий элемент внешнего списка сам является списком `[H | _]`
   - `H` — его голова (первый элемент)
   - `_` — хвост нас не интересует (подчёркивание = игнорируем)
   - Берём `H` в результат, рекурсируем по `Tail`

3. `list_heads([_ | Tail])` — текущий элемент НЕ является непустым списком
   (это атом, число, кортеж `{}`, пустой список `[]` — всё что угодно кроме `[H|_]`)
   — пропускаем, идём дальше

**Важно:** образцы проверяются **сверху вниз**, первый подходящий побеждает.
Образец `[[H|_]|Tail]` проверяется ДО `[_|Tail]`, поэтому непустые списки
обрабатываются правильно.

**Алгоритм для `list_heads([[1,2,3], {true,3}, [4,5], []])`:**
```
[1,2,3] — непустой список, H=1 → [1 | ...]
{true,3} — кортеж, не список → пропуск
[4,5]    — непустой список, H=4 → [4 | ...]
[]       — пустой список, не [H|_] → пропуск
Result: [1, 4]  ✓
```

### Проверка

```erlang
lab2_task1:list_heads([[1,2,3], {true,3}, [4,5], []]). % => [1,4]
lab2_task1:list_heads([a, b, [1], [2,3]]).             % => [1,2]
lab2_task1:list_heads([]).                             % => []
```

## Задание 2: `takewhile/2` — файл `lab2_task2.erl`

### Код

```erlang
takewhile(_Pred, []) ->
    [];
takewhile(Pred, [H | T]) ->
    case Pred(H) of
        true  -> [H | takewhile(Pred, T)];
        false -> []
    end.
```

### Объяснение

**Задача:** вернуть начало списка, пока все элементы удовлетворяют предикату.
Как только первый элемент не прошёл — **немедленно останавливаемся**, остаток
списка не просматриваем вообще.

**Отличие от `filter`:**
- `filter(Pred, List)` — просматривает **весь** список, возвращает все элементы где Pred=true
- `takewhile(Pred, List)` — берёт элементы с начала, стоп при первом false

```
filter(fun(X) -> X < 10 end, [1,3,9,11,6])   => [1,3,9,6]   (6 включён!)
takewhile(fun(X) -> X < 10 end, [1,3,9,11,6]) => [1,3,9]     (6 не проверяется)
```

**Параметр `Pred`** — это анонимная функция, принимающая один элемент и возвращающая
`true` или `false`. Мы вызываем её через `Pred(H)`.

**Алгоритм для `takewhile(fun(X) -> X < 10 end, [1,3,9,11,6])`:**
```
H=1:  Pred(1)  = true  → [1 | takewhile(Pred, [3,9,11,6])]
H=3:  Pred(3)  = true  → [3 | takewhile(Pred, [9,11,6])]
H=9:  Pred(9)  = true  → [9 | takewhile(Pred, [11,6])]
H=11: Pred(11) = false → []   ← СТОП
Result: [1,3,9]  ✓
```

### Проверка

```erlang
lab2_task2:takewhile(fun(X) -> X < 10 end, [1,3,9,11,6]). % => [1,3,9]
lab2_task2:takewhile(fun(X) -> X > 0 end, [1,2,3]).        % => [1,2,3]
lab2_task2:takewhile(fun(X) -> X > 0 end, [-1,2,3]).       % => []
```

## Задание 3: `iterate/2` — файл `lab2_task3.erl`

### Код

```erlang
iterate(F, N) ->
    fun(X) -> apply_n(F, N, X) end.

apply_n(_F, 0, X) -> X;
apply_n(F, N, X)  -> apply_n(F, N - 1, F(X)).
```

### Объяснение

**Задача:** создать функцию, которая применяет `F` к аргументу ровно `N` раз по цепочке.

```
iterate(F, 0)(X) = X
iterate(F, 1)(X) = F(X)
iterate(F, 2)(X) = F(F(X))
iterate(F, 3)(X) = F(F(F(X)))
```

**`iterate/2` возвращает функцию** — это ключевой момент.
`fun(X) -> apply_n(F, N, X) end` — анонимная функция, которая «знает» про `F` и `N`
благодаря замыканию: они захвачены из окружающего контекста.

**`apply_n/3` — хвостовая рекурсия с аккумулятором:**
- База: `apply_n(F, 0, X) = X` — применили 0 раз, возвращаем `X`
- Шаг: применяем `F` к `X`, уменьшаем счётчик, рекурсируем

**Пример `F1 = iterate(fun(X) -> {X} end, 2), F1(1)`:**
```
apply_n(F, 2, 1)
  → apply_n(F, 1, F(1))  = apply_n(F, 1, {1})
  → apply_n(F, 0, F({1})) = apply_n(F, 0, {{1}})
  → {{1}}  ✓
```

### Проверка

```erlang
F1 = lab2_task3:iterate(fun(X) -> {X} end, 2),
F1(1).   % => {{1}}

F2 = lab2_task3:iterate(fun(X) -> X * 2 end, 3),
F2(1).   % => 8  (1→2→4→8)

F3 = lab2_task3:iterate(fun(X) -> X end, 0),
F3(42).  % => 42  (0 применений = тождественная функция)
```

## Задание 4: `integrate/2` — файл `lab2_task4.erl`

### Код

```erlang
integrate(F, N) ->
    fun(A, B) ->
        H = (B - A) / N,
        lists:foldl(
            fun(I, Acc) ->
                MidX = A + (I - 0.5) * H,
                Acc + F(MidX) * H
            end,
            0.0,
            lists:seq(1, N)
        )
    end.
```

### Объяснение

**Задача:** создать функцию, которая приближённо считает определённый интеграл.

**Метод средних прямоугольников:**
Разбиваем отрезок `[A, B]` на `N` одинаковых частей шириной `H = (B-A)/N`.
На каждой части строим прямоугольник высотой `F(середина части)`.
Площадь прямоугольника ≈ площади под кривой на этом участке.

```
∫ₐᵇ F(x) dx ≈ H * Σ F(A + (i - 0.5) * H),  i = 1..N
```

**`lists:foldl(Fun, Acc0, List)`** — встроенная функция свёртки:
- Обходит `List` слева направо
- На каждом шаге: `Acc = Fun(Element, Acc)`
- Возвращает финальный аккумулятор

Это аналог цикла `for` с накоплением суммы.

**`lists:seq(1, N)`** — генерирует список `[1, 2, 3, ..., N]`.

**Пример `integrate(fun(X) -> X end, 100)(0, 1)`:**
Интеграл `∫₀¹ x dx = x²/2 |₀¹ = 0.5`
```
H = 1/100 = 0.01
Сумма: Σ (i-0.5)*0.01 * 0.01, i=1..100  ≈ 0.5  ✓
```
Чем больше N, тем точнее результат.

### Проверка

```erlang
F1 = lab2_task4:integrate(fun(X) -> X end, 100),
F1(0, 1).     % => ~0.5

F2 = lab2_task4:integrate(fun(X) -> X * X end, 1000),
F2(0, 1).     % => ~0.333  (∫₀¹ x² dx = 1/3)

F3 = lab2_task4:integrate(fun(_X) -> 1 end, 10),
F3(0, 5).     % => 5.0  (∫₀⁵ 1 dx = 5, точно)
```

## Задание 5: `for/4` — файл `lab2_task5.erl`

### Код

```erlang
for(I, Cond, Step, Body) ->
    case Cond(I) of
        false -> ok;
        true  ->
            Body(I),
            for(Step(I), Cond, Step, Body)
    end.
```

### Объяснение

**Задача:** воспроизвести поведение C-шного `for`:
```c
for (int i = Init; Cond(i); i = Step(i)) { Body(i); }
```

В Erlang нет переменных (в смысле изменяемых), нет циклов. Заменяем цикл рекурсией,
а изменяемую переменную `i` — аргументом рекурсивной функции.

**Разбор аргументов:**
- `I` — текущее значение (начинается с `Init`)
- `Cond` — `fun(I) -> true | false` — условие продолжения
- `Step` — `fun(I) -> NewI` — вычисляет следующее значение
- `Body` — `fun(I) -> any()` — тело цикла, результат игнорируется

**Шаги выполнения:**
1. Проверяем `Cond(I)`
2. Если `false` → конец, возвращаем `ok`
3. Если `true` → выполняем `Body(I)`, вычисляем `Step(I)`, рекурсируем

**Это хвостовая рекурсия** (рекурсивный вызов — последнее действие), поэтому
стек не растёт даже для миллиона итераций.

**Пример — напечатать чётные числа от 2 до 10:**
```erlang
lab2_task5:for(
    2,
    fun(I) -> I =< 10 end,
    fun(I) -> I + 2 end,
    fun(I) -> io:format("~p~n", [I]) end
).
% Выведет: 2 4 6 8 10
```

### Проверка

```erlang
% Числа 1..5
lab2_task5:for(1, fun(I) -> I =< 5 end, fun(I) -> I+1 end,
    fun(I) -> io:format("~p ", [I]) end).
% => 1 2 3 4 5

% Убывающий счётчик
lab2_task5:for(5, fun(I) -> I > 0 end, fun(I) -> I-1 end,
    fun(I) -> io:format("~p ", [I]) end).
% => 5 4 3 2 1

% Накопление суммы через process dictionary
put(s, 0),
lab2_task5:for(1, fun(I) -> I =< 10 end, fun(I) -> I+1 end,
    fun(I) -> put(s, get(s) + I) end),
get(s).   % => 55
```

## Задание 6: `sort_by/2` — файл `lab2_task6.erl`

### Код

```erlang
sort_by(_Cmp, [])  -> [];
sort_by(_Cmp, [X]) -> [X];
sort_by(Cmp, List) ->
    {Left, Right} = split_half(List),
    SortedLeft    = sort_by(Cmp, Left),
    SortedRight   = sort_by(Cmp, Right),
    merge(Cmp, SortedLeft, SortedRight).

split_half(List) ->
    Half = length(List) div 2,
    lists:split(Half, List).

merge(_Cmp, [], Right) -> Right;
merge(_Cmp, Left, []) -> Left;
merge(Cmp, [LH|LT]=Left, [RH|RT]=Right) ->
    case Cmp(LH, RH) of
        less    -> [LH | merge(Cmp, LT, Right)];
        equal   -> [LH | merge(Cmp, LT, Right)];
        greater -> [RH | merge(Cmp, Left, RT)]
    end.
```

### Объяснение

**Задача:** отсортировать список, используя функцию-компаратор.

**Алгоритм: сортировка слиянием (Merge Sort)**  
Выбрана потому что идеально подходит для функциональных языков:
не нужен случайный доступ по индексу, только разбиение и слияние списков.

**Три фазы:**

**1. База рекурсии:**
- Пустой список — уже отсортирован
- Список из одного элемента — уже отсортирован

**2. Разбиение (`split_half`):**
Делим список пополам через `lists:split(Half, List)`.
```
[3,1,4,1,5,9,2]  →  [3,1,4] + [1,5,9,2]  (Half = 7 div 2 = 3)
```

**3. Рекурсивная сортировка + слияние (`merge`):**
Рекурсивно сортируем каждую половину, затем сливаем два отсортированных списка.

**Слияние** — ключевая операция. Берём головы обоих списков, применяем компаратор,
меньший элемент идёт в результат первым, рекурсируем:
```
merge([1,3,4], [1,2,5,9]):
  Cmp(1,1)=equal  → [1 | merge([3,4],[1,2,5,9])]
  Cmp(3,1)=greater → [1 | merge([3,4],[2,5,9])]
  Cmp(3,2)=greater → [2 | merge([3,4],[5,9])]
  Cmp(3,5)=less   → [3 | merge([4],[5,9])]
  Cmp(4,5)=less   → [4 | merge([],[5,9])]
  Left=[]: → [5,9]
Result: [1,1,2,3,4,5,9]  ✓
```

**Компаратор** — функция `fun(X, Y) -> less | equal | greater end`.
Меняя его, можно сортировать по-разному:
```erlang
% По возрастанию (стандартный порядок)
Asc  = fun(X,Y) -> if X<Y->less; X=:=Y->equal; true->greater end end.

% По убыванию
Desc = fun(X,Y) -> if X>Y->less; X=:=Y->equal; true->greater end end.

% Строки по длине
ByLen = fun(X,Y) ->
    LX=length(X), LY=length(Y),
    if LX<LY->less; LX=:=LY->equal; true->greater end
end.
```

**Сложность:** O(n log n) — оптимально для сортировок основанных на сравнениях.

### Проверка

```erlang
Asc = fun(X,Y) -> if X<Y->less; X=:=Y->equal; true->greater end end,
lab2_task6:sort_by(Asc, [3,1,4,1,5,9,2]).  % => [1,1,2,3,4,5,9]

Desc = fun(X,Y) -> if X>Y->less; X=:=Y->equal; true->greater end end,
lab2_task6:sort_by(Desc, [3,1,4,1,5]).     % => [5,4,3,1,1]
```
